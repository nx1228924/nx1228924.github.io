The First Blood--我的第一篇博客

分类：
标签： Hexo

日期：2023-08-27 00:00:00

第一篇博客拖了好久。本来应该是今年一月份，刚创建博客的时候就发的，结果到了现在才发。

这是我创建的第二个博客，先前创建的博客出于一些考虑而丢弃了（简单说，就是想改网址，但又不想绑定域名。而之前忘记了怎么改网址，也没有去Google，因此重新二创博客。

本来还要继续拖着写博客的，最近一位故友问我，怎么我的博客建了不写。我有些不好意思，感觉更多的是对不住自己，有时会有一些三分钟热度，除非是自己坚持要去做的（而实际上，也并非是一时兴起想建个人网站，只是说当时被一些事情给影响了，注意力被迫转移。不过，并未忘记这一茬）。

说明

这篇博客出于以下考虑而写：

*1. 个人开始写博客的心路历程；

  1. 二次搭建网站的感受；
  2. 重述搭建网站的阻碍，复盘。*

利用GitHub和Hexo建站

P：根据现有理解进行个人解释，或有不全面和错误之处。

实际建站可详细参考B站、CSDN等平台提供的方法，Hexo官网也有建立个人网站的教程。

GitHub可以将本地文件上传，在GitHub上能与更多的人交流（联网访问）。Hexo是一个做好的静态网页，对网站进行渲染（美化），本地网站做好了以后可以直接用Hexo离线查看，不过只能自己看。要实现让其他人也能访问自己的网站，需要将网站部署到GitHub。

简单介绍Git

Git是分布式版本控制系统，可以快速高效地进行项目版本管理（还是前面的那位故友跟我提了一下，我才知道这个东西到底是干嘛的，第一次建站的时候不太明白。看完了故友发来的Git资料，才对其有些了解）。

Git可以记录你每次文件的修改变化，每次修改后的文件均有一个专属编码，方便回看旧文件或是恢复误删文件。而GitHub可以简单理解为Git的远程仓库，能将本地文件“备份”到上面，而不用占用内存。【不过GitHub是国外网站，在国内访问速度较慢，有魔法时，访问才比较快。可以使用Gitee代替（国内网站），它的功能和GitHub类似】

插曲

电脑需要安装:node.js、git、hexo 。
由于我之前曾经安装过博客，均已下载，按win+r，输入cmd后进入命令板，再输入node -v、hexo -v和git --version后出现版本号，确认软件安装成功。

  • 因为原本旧的Git和Hexo相关文件并未删除，导致一个Git里面还有一个Git，像套娃一样。而且我两个Git文件都作了修改，已经分不清楚哪个是最新的，当我把密钥弄完，准备将本地文件上传到GitHub上时，系统说我的GitHub远程仓库不存在，我找遍了网上所有能找的方法，都没找到，之后把Git和Hexo文件都删了，重装以后，就可以了🤔

想来，应该是因为我把第一次弄的GitHub仓库删了，又重建了新仓库，新仓库虽然在全局进行了user.name 和 user.email重置，可能因为我前后两次建站共用了一个邮箱，还把两个Git文件进行了不同程度的修改，导致系统识别错误【具体原因不清楚，不确定】。

  • 其他部分则未遇到问题，跟着操作走就是了。

建站

  1. 环境搭建
  • 在本地安装Node.js 和Git 。
  • 注意安装适合自己电脑版本的（以自身为例，我安装的是win10.64bit的），可以在电脑设置“系统”中查看“关于”界面，里面有电脑配置。

2.连接Github

  • 使用邮箱注册Github账户后，
  • 鼠标右键->
    设置用户名和邮箱

git config –global user.name “GitHub 用户名”
git config –global user.email “GitHub 邮箱”

  • 创建SSH密钥：
    依然在git操作面板输入ssh-keygen -t rsa -C "GitHub 邮箱"
    【“邮箱”是指输入自己的邮箱号】

  • 添加密钥：
    进入[C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

登录 Github ，点击 Settings ，选择左侧栏的 SSH and GPG keys ，点击 New SSH key 。

名字自取，粘贴复制的 id_rsa.pub 内容到 Key 中，点击添加即可。

  • 验证连接：
    打开 Git Bash ，输入ssh -T git@github.com，若操作面板询问是否确定【Are you sure……】，输入yes即可。

当电脑屏幕显示你已经成功安装就可以了，不用管后面的 “but—-“ 内容。

3.在 Github 上创建仓库
GitHub 主界面有个加号，点它就对了【New repository】

创建好后，博客地址为https://用户名.github.io
【用户名可以自己改，就是你创建仓库的 name 】
